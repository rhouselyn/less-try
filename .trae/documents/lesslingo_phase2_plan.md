# 少邻国 (Lesslingo) - 阶段二开发计划

## 项目现状分析

### 已完成功能
- 前端基本框架（输入、字典、学习三个步骤）
- 后端API（文本处理、单词生成、学习进度管理）
- NVIDIA API集成（分词翻译、生成选择题）
- 基本的单词学习功能（选择题形式）

### 待实现功能
1. **前端重构**：将App.jsx分散到多个文件
2. **单词分组**：实现10个单词一组的分组逻辑
3. **进度页面**：实现进度显示页面，每个单元有一个按钮
4. **CheckCoverage算法**：检查已学单词能否组成新句子
5. **句子翻译题**：实现句子翻译题，包括冗余tokens
6. **API修改**：修改NVIDIA API的tool定义，在处理句子时生成4个冗余tokens

## 开发计划

### 阶段一：前端重构

#### 任务1：拆分App.jsx
- 将App.jsx拆分为多个组件文件
- 建立合理的目录结构
- 保持功能不变，仅优化代码结构

#### 任务2：创建进度页面组件
- 实现进度显示页面
- 每个单元显示一个按钮
- 已学习过的单元可以重复学习，没学过的无法点击
- 点击进度后进入单词学习页面

### 阶段二：后端功能增强

#### 任务3：实现单词分组逻辑
- 在后端实现10个单词一组的分组
- 为每个文件生成固定的分组结构
- 保存分组信息到本地存储

#### 任务4：实现CheckCoverage算法
- 编写算法检查已学单词能否组成原文中还没练过的句子
- 实现动态循环测试逻辑
- 当能组成新句子时，插入翻译题

#### 任务5：修改NVIDIA API tool定义
- 修改split_and_translate tool，在处理句子时生成4个冗余tokens
- 确保冗余tokens与原文相关
- 更新相关数据结构

### 阶段三：前端功能实现

#### 任务6：实现句子翻译题
- 前端实现句子翻译题界面
- 支持点击token组成顺序正确的翻译
- 显示正确答案和错误提示

#### 任务7：实现学习流程
- 实现从单词学习到句子翻译题的流程
- 实现单元结束后的导航逻辑
- 实现返回单词表的功能

### 阶段四：测试与优化

#### 任务8：功能测试
- 测试单词分组功能
- 测试CheckCoverage算法
- 测试句子翻译题
- 测试学习流程

#### 任务9：性能优化
- 优化前端渲染性能
- 优化后端API响应速度
- 优化缓存机制

### 后端API扩展
- `/api/learn/{file_id}/progress`：获取学习进度和分组信息
- `/api/learn/{file_id}/unit/{unit_id}`：获取指定单元的单词
- `/api/learn/{file_id}/check-coverage`：检查已学单词能否组成新句子
- `/api/learn/{file_id}/sentence-quiz`：生成句子翻译题

### 数据结构扩展
- `pipeline_data.json`：添加`redundant_tokens`字段
- `progress/state.json`：添加分组学习进度

## 风险与应对策略

### 风险1：API限速
- 应对策略：保持TTS间隔1-2秒随机时长
- 实现断点续传机制

### 风险2：题目漂移
- 应对策略：所有LLM生成的内容存入snapshot.json
- 严格读取快照数据

### 风险3：性能问题
- 应对策略：优化前端渲染
- 实现缓存机制
- 异步处理耗时操作
