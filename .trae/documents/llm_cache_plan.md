# LLM生成结果缓存计划

## 问题分析

用户反馈了以下需求：
1. **避免重复生成**：LLM生成过的单词释义不需要重复生成
2. **缓存机制**：如果在单词表中生成过了，就记录下来，随机练习或下次点击到了就不用重复生成
3. **温度设置**：LLM需要使用temp=0，确保生成结果的一致性
4. **固定随机种子**：确保每次随机的顺序一样
5. **后台提前生成**：在生成当前单词表的时候在后台提前生成下一个单词，减少等待时间

## 解决方案

### 1. 实现单词信息缓存机制
- 在后端添加缓存存储，记录已生成的单词信息
- 每次请求单词信息前先检查缓存
- 只有在缓存中不存在时才调用LLM生成

### 2. 确保temp=0设置
- 检查并确保所有LLM调用都使用temp=0
- 验证生成结果的一致性

### 3. 固定随机种子
- 在随机单词选择时使用固定种子
- 确保每次生成的随机顺序一致

### 4. 后台提前生成
- 在学习模式中，当前单词显示时，后台异步生成下一个单词
- 预加载下一个单词的信息，减少用户等待时间

### 5. 缓存存储设计
- 使用文件系统或内存存储缓存
- 按文件ID和单词进行索引
- 缓存包含完整的单词信息（释义、音标、例句、变体、记忆提示等）

## 预期结果
- 单词信息只生成一次
- 重复请求从缓存中获取，提高响应速度

- 每次随机顺序一致

## 风险与应对
- **缓存过期**：考虑添加缓存过期机制，避免缓存数据过时
- **内存使用**：对于大量单词，考虑使用文件系统存储而非内存
- **一致性**：确保缓存与原始数据的一致性
- **并发处理**：确保后台预生成不会影响主流程
