# LLM生成结果缓存计划

## 问题分析

用户反馈了以下需求：
1. **避免重复生成**：LLM生成过的单词释义不需要重复生成
2. **缓存机制**：如果在单词表中生成过了，就记录下来，随机练习或下次点击到了就不用重复生成
3. **温度设置**：LLM需要使用temp=0，确保生成结果的一致性

## 解决方案

### 1. 实现单词信息缓存机制
- 在后端添加缓存存储，记录已生成的单词信息
- 每次请求单词信息前先检查缓存
- 只有在缓存中不存在时才调用LLM生成

### 2. 确保temp=0设置
- 检查并确保所有LLM调用都使用temp=0
- 验证生成结果的一致性

### 3. 缓存存储设计
- 使用文件系统或内存存储缓存
- 按文件ID和单词进行索引
- 缓存包含完整的单词信息（释义、音标、例句、变体、记忆提示等）

   - 确定缓存的存储方式和结构

3. **实现缓存功能**：
   - 在`get_word_details`和`get_random_word`端点添加缓存检查
   - 实现缓存的存储和读取

4. **验证temp=0设置**：
   - 检查所有LLM调用的温度参数
   - 确保生成结果的一致性

5. **测试功能**：

## 预期结果
- 单词信息只生成一次
- 重复请求从缓存中获取，提高响应速度
- LLM生成结果一致（temp=0）
- 学习模式和单词详情查看都能使用缓存

## 风险与应对
- **缓存过期**：考虑添加缓存过期机制，避免缓存数据过时
- **内存使用**：对于大量单词，考虑使用文件系统存储而非内存
- **一致性**：确保缓存与原始数据的一致性

## 验证方法
- 检查API响应时间，验证缓存是否生效
- 查看日志，确认LLM只在缓存未命中时调用
- 验证生成结果的一致性
- 测试学习模式的流畅度
