# Lesslingo 全面修复和优化计划

## 问题分析

根据用户反馈和代码审查，以下问题需要修复：

### 1. 语言硬编码问题
- `main.py` 中的 `get_word_details` 函数仍有硬编码 `target_lang = "zh"`
- 所有地方都应该使用从用户选择并存储的语言设置

### 2. 文本检测和自动翻译功能
- 需要在提示词中添加逻辑：如果输入文本不是学习语言，先自动翻译成学习语言，再生成母语翻译
- 直接在 `nvidia_api.py` 的提示词中处理，无需单独的语言检测步骤

### 3. 提示词需要优化
- 确保 `nvidia_api.py` 中的提示词能正确处理各种语言组合
- 特别是学习语言为中文、母语为英文的情况
- 在提示词中明确说明：如果当前文本不是学习语言，先严格翻译成学习语言

### 4. 单词重复问题
- 已经添加了固定随机种子和学习进度保存
- 需要确保没有重复索引的问题

### 5. 空格处理问题
- 已经在 `text_processor.py` 中移除了过度的空格处理
- 需要确保LLM生成自然的翻译结果
