# 全面bug修复和功能增强计划

## 问题分析

用户反馈了以下问题和需求：

### 1. 语言处理问题
- 当学习语是中文，母语是英文时功能需要正确
- 如果粘贴的文本不是需要学习的语言，LLM先自动翻译成该语言，然后再生成选定母语的翻译
- 应该调用选择的状态而不是直接写死

### 2. 代码问题
- 不应该有去除空格这类硬性操作，应该直接由LLM生成
- 遗留的bug

### 3. 功能问题
- 测试中连续出现两个english，可能是随机种子或查重问题
- 需要完整测试所有按钮

### 4. 已发现的问题
- `main.py`中第245、342、420行硬编码了`target_lang = "zh"`
- `text_processor.py`中过度处理了翻译结果，强制移除了空格
- 随机单词选择逻辑有问题，可能导致重复选择
