# 文本处理流程说明

## 1. 整体流程概述

当用户输入文本后，系统会按照以下流程处理：

1. 前端发送文本到后端 API
2. 后端接收文本，分割成句子
3. 对每个句子调用 LLM 进行翻译和处理
4. 提取单词并创建字典条目
5. 将结果保存并返回给前端
6. 前端轮询状态，实时显示处理进度

## 2. 详细流程

### 2.1 前端处理流程

#### 步骤 1: 用户输入
- 用户在输入页面输入文本
- 选择源语言和目标语言
- 点击"开始处理"按钮

#### 步骤 2: API 请求
- 前端调用 `/api/process-text` 端点
- 发送文本、源语言、目标语言

#### 步骤 3: 轮询状态
- 前端立即跳转到单词表页面
- 开始轮询 `/api/status/{file_id}` 端点
- 每 500ms 检查一次状态，最多 300 次（10 分钟）

#### 步骤 4: 实时更新
- 显示处理进度百分比
- 显示当前处理的句子序号/总句子数
- 实时更新单词表和句子翻译

#### 步骤 5: 完成处理
- 当状态变为 "completed" 时停止轮询
- 显示完整的单词表和句子翻译
- 用户可以开始学习

### 2.2 后端处理流程

#### 步骤 1: 接收请求
- 接收来自前端的文本处理请求
- 生成唯一的 `file_id`
- 立即返回 `file_id`，后台开始处理

#### 步骤 2: 文本分割
- 调用 `text_processor.split_sentences()` 将文本分割成句子
- 句子结束符包括：`.`, `!`, `?`, `。`, `！`, `？`
- 保留原始空格

#### 步骤 3: 句子处理循环
对每个句子进行以下处理：

1. **LLM 调用**
   - 调用 `text_processor.process_translation()`
   - 使用 LLM 对句子进行翻译
   - 同时生成单词的字典条目

2. **进度更新**
   - 更新处理状态
   - 计算当前进度百分比
   - 保存到 `processing_status` 中

#### 步骤 4: 字典提取与合并
- 从所有句子的翻译结果中提取字典条目
- 对字典条目进行去重
- 按字母顺序排序单词

#### 步骤 5: 数据保存
- 保存句子翻译
- 保存单词表
- 保存打乱的单词顺序（用于学习）

#### 步骤 6: 预生成
- 后台预生成第一个单词的详细信息
- 提升用户体验

### 2.3 LLM 处理流程

当处理单个句子时，LLM 会生成：

1. **原始句子** - 保留原始文本
2. **翻译 token 数组** - 每个单词的翻译信息
3. **完整翻译** - 自然流畅的完整翻译
4. **语法说明** - 对整个句子的语法解释
5. **冗余 tokens** - 用于测试的干扰项
6. **字典条目** - 句子中每个单词的详细信息

字典条目包含：
- 单词本身
- 音标
- 上下文意思
- 词形变化
- 例句
- 选项（用于选择题）
- 语法说明
- tokens 列表
- 词性

## 3. 前端显示流程

### 3.1 处理中的显示
- 显示进度条
- 显示当前处理的句子
- 实时更新单词表
- 实时更新句子列表

### 3.2 处理完成后的显示
- 显示完整的单词表
- 显示句子列表
- 可以点击单词查看详情
- 可以点击句子查看语法说明

### 3.3 学习流程
1. 进入阶段选择页面（将改为直接显示所有单元）
2. 选择单元开始学习
3. 进行单词练习
4. 进行翻译练习
5. 完成单元

## 4. 数据流架构

```
用户输入
   ↓
前端 (React)
   ↓
POST /api/process-text
   ↓
后端 FastAPI
   ↓
text_processor.split_sentences()
   ↓
循环处理每个句子
   ↓
text_processor.process_translation()
   ↓
Nvidia API (LLM)
   ↓
提取字典条目
   ↓
去重、排序
   ↓
保存数据
   ↓
更新处理状态
   ↓
前端轮询 GET /api/status/{file_id}
   ↓
实时显示
```

## 5. 关键技术点

### 5.1 异步处理
- 使用 FastAPI 的 `BackgroundTasks` 进行后台处理
- 避免长时间阻塞 HTTP 请求
- 立即返回响应，后台继续工作

### 5.2 状态管理
- 使用内存字典 `processing_status` 存储处理状态
- 支持实时进度更新
- 支持错误状态记录

### 5.3 轮询机制
- 前端定期检查状态
- 处理完成自动停止轮询
- 超时处理和错误恢复

### 5.4 LLM 调用
- 使用 NVIDIA API 进行翻译和字典生成
- 调用超时设置为 10 分钟
- 支持重试机制

## 6. 错误处理

### 6.1 后端错误
- 处理过程中出现错误会捕获并记录
- 更新状态为 "error"
- 错误信息返回给前端

### 6.2 前端错误
- 网络错误会显示提示
- 处理超时会提示重试
- API 错误会显示具体信息
