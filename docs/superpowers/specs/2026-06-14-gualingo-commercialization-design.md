# 呱邻国商业化设计方案

**日期：** 2026-06-14
**状态：** 已确认
**作者：** 产品讨论

---

## 1. 背景与目标

呱邻国（Gualingo）是一个 AI 驱动的沉浸式外语学习平台，用户输入任意文本，AI 自动生成词汇表、分句翻译和多种练习题。支持任意语言互学，当前仅需一个 LLM API Key 即可使用。

**商业化目标：** 将呱邻国从开源工具转型为 Freemium SaaS 产品，实现可持续收入。

**目标用户：**
- 优先：学生群体（中学/大学，阅读理解、考试备考）
- 其次：旅游人士（实用场景对话学习）
- 远期：教育机构/B 端

**当前状态：** 尚未正式发布，处于内测阶段。核心挑战是产品不够成熟。

---

## 2. 商业模式：Freemium SaaS

### 2.1 双版本策略

```
┌─────────────────────────────────────────────┐
│           呱邻国 Community (开源)             │
│  BYOK · 本地存储 · 基础学习功能 · 多Key轮询    │
│  许可证: AGPL v3 (确保贡献回馈)                │
└──────────────────────┬──────────────────────┘
                       │ 共享核心代码
┌──────────────────────┴──────────────────────┐
│           呱邻国 Pro (闭源 SaaS)              │
│  平台API · 云同步 · SRS · 分析 · AI口语       │
│  用户系统 · 支付 · 移动端 · 社交              │
└─────────────────────────────────────────────┘
```

**社区版定位：** 不是"阉割版"，而是"自助版"。功能完整，用户自己管 API Key 和数据。适合技术用户和自建部署者，是最好的口碑传播渠道和社区贡献来源。

**商业版价值：** 便利（云同步、平台 API）+ 效果（SRS、AI 口语、学习分析），不是功能封锁。

### 2.2 付费层级

| 层级 | 月费 | 包含内容 |
|------|------|---------|
| **免费版** | ¥0 | BYOK，本地存储，基础学习功能（社区版） |
| **基础版** | ¥19-29 | 平台 API 额度（月 50 次文本处理），云同步，SRS 间隔复习 |
| **专业版** | ¥49-69 | 无限 API 额度，AI 口语对话，学习分析，优先支持 |

### 2.3 核心变现逻辑

1. **API 代充：** 批量采购 LLM API（SiliconFlow/DeepSeek 等），加价 30-50% 卖给用户
2. **云服务：** 云同步、跨设备、数据备份——用户为便利付费
3. **高级功能：** SRS、AI 口语、分析——用户为效果付费

## 3. 功能评估

### 3.1 保留的核心功能

| 功能 | 理由 |
|------|------|
| 三种输入模式（直接/翻译/生成） | 产品最独特的价值 |
| AI 分句翻译 + 词汇提取 | 核心能力，不可替代 |
| 两阶段学习 + 5 种题型 | 学习体系完整，差异化优势 |
| 词汇表/字典浏览 | 学习过程的核心参考 |
| 错题回顾 | 学习效果的关键保障 |
| TTS 语音朗读 | 语言学习必备 |
| 多语言 UI | 国际化基础 |
| 历史记录 | 用户留存的基础 |

### 3.2 删除/简化

| 项目 | 原因 | 处理方式 |
|------|------|---------|
| `Guapage/` 目录 | 旧版/未使用的前端 | 删除 |
| 双存储系统（文件 + SQLite） | 过度工程，维护成本高 | 统一为 SQLite，移除文件存储和双写逻辑 |
| GPL v3 许可证 | 商业化障碍 | 社区版改 AGPL v3，商业版闭源 |

### 3.3 商业化必须新增

| 功能 | 优先级 | 理由 |
|------|--------|------|
| 用户注册/登录系统 | P0 | SaaS 的基础 |
| 云同步（学习进度跨设备） | P0 | 付费用户最基本的需求 |
| 平台 API 代理服务 | P0 | 核心变现手段 |
| 支付系统（订阅/按量） | P0 | 收钱的基础 |
| SRS 间隔重复算法 | P1 | 语言学习的科学方法，付费版核心卖点 |
| 学习数据分析仪表盘 | P1 | 让用户看到进步，提升续费率 |
| AI 口语对话练习 | P1 | 高价值差异化功能，适合专业版 |
| 移动端适配/PWA | P1 | 学生群体手机使用率极高 |
| 内容社区/课程市场 | P2 | 增加用户粘性和内容供给 |
| 社交功能（排行榜/学习小组） | P2 | 提升活跃度和传播 |

## 4. 产品路线图

### Phase 0：商业化准备（上线前必做）

**目标：** 让产品达到"可以收费"的最低标准

| 任务 | 说明 |
|------|------|
| 用户注册/登录 | 邮箱 + Google/GitHub OAuth，JWT 鉴权 |
| 平台 API 代理 | 统一代理 LLM 调用，按用户配额限流 |
| 云同步基础 | 学习进度、词汇表、历史记录同步到云端 |
| 支付集成 | Stripe（海外）+ 微信/支付宝（国内） |
| 许可证变更 | 社区版改 AGPL v3，商业版代码独立仓库 |
| 代码清理 | 删除 Guapage/，统一存储为 SQLite |

**交付物：** 可注册、可付费、数据可云同步的 SaaS 版本

### Phase 1：付费价值验证（上线后 1-3 个月）

**目标：** 验证用户愿意为哪些功能付费

| 任务 | 说明 |
|------|------|
| SRS 间隔重复 | 基于遗忘曲线的智能复习调度，专业版独占 |
| 学习数据分析 | 每日/每周学习统计、词汇掌握率、学习趋势 |
| 移动端适配 | PWA 或响应式优化，让学生手机上也能用 |
| API 用量仪表盘 | 让用户看到自己的 API 消耗，建立用量感知 |

**关键指标：** 免费转付费率 > 5%，月留存 > 40%

### Phase 2：差异化壁垒（3-6 个月）

**目标：** 建立竞争壁垒，提升 ARPU

| 任务 | 说明 |
|------|------|
| AI 口语对话 | 语音识别 + LLM 对话，模拟真实对话场景 |
| 课程/内容市场 | 预设学习包（JLPT、TOEFL、旅游日语等） |
| 社交功能 | 学习排行榜、学习小组、打卡分享 |
| 浏览器扩展 | 一键从网页抓取文本到呱邻国学习 |

**关键指标：** 专业版占比 > 30%，月活 > 10000

### Phase 3：B 端拓展（6-12 个月）

**目标：** 打开教育机构市场

| 任务 | 说明 |
|------|------|
| 教师管理后台 | 班级管理、作业布置、学习进度监控 |
| 机构版定价 | 按座位数计费，批量授权 |
| 白标方案 | 允许机构定制品牌和内容 |
| API 开放平台 | 允许第三方集成呱邻国的学习引擎 |

## 5. Phase 0 技术改造要点

### 5.1 用户系统

```
注册方式: 邮箱 + 密码 / Google OAuth / GitHub OAuth
鉴权: JWT (access_token 15min + refresh_token 7天)
数据模型:
  - users: id, email, name, password_hash, oauth_provider, tier, created_at
  - subscriptions: user_id, plan, status, stripe_customer_id, current_period_end
```

### 5.2 API 代理层

```
架构: 在现有 nvidia_api.py 上封装一层
逻辑:
  - BYOK 用户: 直接用用户自己的 Key（免费版）
  - 平台代理用户: 从平台配额池中分配 Key，按用户限流
  - 配额管理: Redis 计数器，按月重置
  - 成本控制: 每用户每日/每月调用上限，超限降级提示
```

### 5.3 云同步

```
策略: 增量同步，非全量覆盖
数据模型:
  - 云端存储: PostgreSQL (用户数据) + 当前 SQLite (本地缓存)
  - 同步协议: 客户端上报本地变更 → 服务端合并 → 下发最新状态
  - 冲突处理: 最后写入胜出 (LWW)，简单可靠
  - 离线支持: 本地优先写入，上线后自动同步
```

### 5.4 支付系统

```
海外: Stripe (订阅管理 + 自动续费 + 发票)
国内: 微信支付 + 支付宝 (通过聚合支付如 Ping++ 或直接对接)
流程: 用户选择计划 → 跳转支付 → 回调确认 → 更新用户 tier → 开通权限
```

### 5.5 代码架构调整

```
gualingo-core/          ← 开源社区版（AGPL v3）
  ├── backend/          ← 现有后端，保留 BYOK + 多Key轮询
  ├── frontend/         ← 现有前端
  └── desktop/          ← 桌面端

gualingo-cloud/         ← 闭源商业版
  ├── auth/             ← 用户认证服务
  ├── api-proxy/        ← API 代理层
  ├── sync/             ← 云同步服务
  ├── billing/          ← 支付服务
  └── web/              ← SaaS Web 前端（基于社区版扩展）
```

### 5.6 需要删除的

| 项目 | 原因 |
|------|------|
| `Guapage/` 目录 | 旧版前端，未使用 |
| `storage.py` 文件存储 | 统一用 `db_storage.py`（SQLite），移除双写逻辑 |
| `backend/.env.example` 中的硬编码默认模型 | 改为动态配置 |

## 6. 风险与应对

| 风险 | 应对 |
|------|------|
| API 成本不可控 | 设置每用户每日/每月上限，超限降级；批量采购压低单价 |
| 社区版分流付费用户 | 社区版定位"自助"，商业版卖"便利+效果"；AGPL 防止闭源分支 |
| 多邻国等大平台竞争 | 差异化：任意文本学习、任意语言对、AI 生成内容 |
| 用户不愿付费 | 先验证 PMF，确保免费版足够好用以获客，付费版有明确升级动力 |
| LLM 质量不稳定 | 支持多模型切换，建立质量评估机制，缓存优质结果 |
